# Intervention Template Clusters — Phase 1 Seed Mapping

> **Purpose:** show which scenarios from `intervention-scenarios.md` map to which seeded `InterventionTemplate` row.
>
> **Why cluster?** Many scenarios share the underlying intervention pattern. A "weary arrival" template covers the missed-connection flight, the rain-soaked traveler, the funeral guest, the cancer-recovery family — they all need: skip small talk, offer comfort, give the room time, communicate ETA precisely. Coding 200 individual templates would mean 200 rows of nearly-identical metadata. Coding 10 patterns means the system can match a new scenario by signal pattern + affect quadrant rather than by exact-string matching.
>
> **Phase 1 Selection Criterion:** ten templates that span all four affect quadrants, cover the demo's plausible moments at a Grand Floridian-shaped property, and exercise every approach class (active-waiting, gentle-approach, protective, cascade-only).

## How a future operator extends this

When the network scales (master doc §10.7):

1. New scenario surfaces in the field. Operator adds it to `intervention-scenarios.md` with the next number.
2. Operator either maps it to an existing cluster (most common path) OR proposes a new cluster.
3. New clusters become new templates via SQL migration `02N_seed_intervention_<pattern>.sql`.
4. Echo-Deep (Phase 3) eventually learns to cluster automatically as the corpus grows — but the operator-curated clusters remain the canonical source.

The 200 scenarios are the floor. The system gets better as more land.
