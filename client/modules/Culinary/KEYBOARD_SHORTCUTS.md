## Tips for Using Shortcuts

### Keyboard Shortcut Modifiers

- **Ctrl** on Windows/Linux
- **⌘** (Command) on Mac
- **Shift** for advanced options
- Combine with other keys for specific actions

### When Shortcuts Don't Work

1. **Check Focus**: Make sure the browser window is in focus (not another application)
2. **Input Fields**: Some shortcuts may be disabled while typing in form fields
3. **Browser Conflicts**: Some keyboard shortcuts may conflict with browser defaults
   - Example: **Ctrl+S** opens browser Save dialog (override in preferences if needed)
4. **Non-US Keyboards**: Shortcut display shows US keyboard layout; adjust based on your layout

## Troubleshooting

### "Shortcut isn't working"

1. Check that you're using the correct modifier key (Ctrl on Windows/Linux, ⌘ on Mac)
2. Verify that the focus is on the EchoRecipePro window, not another application
3. Some form inputs may block shortcuts temporarily
4. Try refreshing the page if shortcuts become unresponsive

### "Keyboard layout issue"

If your keyboard layout is non-US:
- Shortcuts are displayed for US QWERTY layout
- Your physical keys might produce different characters
- Check your system keyboard settings if experiencing issues

### "Browser shortcut conflict"

Some browsers reserve certain shortcuts:
- **Ctrl+S**: Browser "Save Page" (use Ctrl+Shift+S as workaround)
- **Ctrl+P**: Browser "Print" (use application menu instead)
- **Ctrl+N**: Browser "New Window" (depends on browser)

## Accessibility

All shortcuts are optional. EchoRecipePro supports:

- **Tab Navigation**: Navigate using Tab and Shift+Tab keys
- **Arrow Keys**: Move between items in lists and menus
- **Enter Key**: Activate buttons and links
- **Space Bar**: Toggle checkboxes and radio buttons
- **Escape Key**: Close dialogs and cancel operations

For full accessibility support, see [Accessibility Documentation](./docs/accessibility.md) (if available).
