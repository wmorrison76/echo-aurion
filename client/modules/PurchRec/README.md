# PurchRec Module (Purchasing Ôćĺ Receiving)

This module hosts the end-to-end purchasing to inventory MVP used by Echo Recipe Pro and DishAssembly.

## Environment Flags

Add the following to your environment (e.g. `.env.local` or Builder project settings):

```
VITE_LUCCCA_ENV=dev
VITE_PURCHREC_PUNCHOUT_ENABLED=false
VITE_PURCHREC_TAX_DEFAULT=0.00
VITE_PURCHREC_FREIGHT_DEFAULT=0.00
```

## Directory Overview

- `api/` ÔÇô Facade consumed by the host app (`purchrec.client.ts`).
- `components/` ÔÇô React panels (Order Guide, Order Form, Receiving, Inventory views).
- `data/` ÔÇô Zod schemas and fixtures used for local development.
- `hooks/` ÔÇô React hooks wrapping services (`useOrderGuide`, `usePurchaseOrder`, etc.).
- `services/` ÔÇô Pure logic (order guide assembly, purchase orders, receiving, inventory adapters).
- `state/` ÔÇô Local Zustand store for PurchRec UI.
- `utils/` ÔÇô Shared helper utilities (cost math, vendor helpers, id generation).

## UI Surface

| Component            | Description                                                                                                                                                    |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OrderGuidePanel`    | Builds a suggestive order guide from menu demand, par levels, and vendor catalogs. Emits selected `OrderGuideRow[]` for the order form.                        |
| `OrderFormDrawer`    | Full-width drawer for drafting, updating, and submitting purchase orders created from the guide or manually. Uses `usePurchaseOrder()`.                        |
| `ReceivingPanel`     | Drives the receiving workflow by matching deliveries to submitted POs, capturing per-line receipts, and generating lots/ledger entries. Uses `useReceiving()`. |
| `InventoryLotsPanel` | Presents current inventory lots with filtering by vendor, text search, and expiry signals. Leverages `useInventory()` for lot data.                            |
| `StockLedgerPanel`   | Audits stock transactions (receipts/issues/adjustments) with type/date filters and summary metrics. Also uses `useInventory()`.                                |

```tsx
import { OrderGuidePanel } from "@/modules/PurchRec/components/OrderGuidePanel";
import { OrderFormDrawer } from "@/modules/PurchRec/components/OrderFormDrawer";
import { ReceivingPanel } from "@/modules/PurchRec/components/ReceivingPanel";
import { InventoryLotsPanel } from "@/modules/PurchRec/components/InventoryLotsPanel";
import { StockLedgerPanel } from "@/modules/PurchRec/components/StockLedgerPanel";
```

Each panel accepts the following shared props:

- `panelId`: string displayed in the header to help end-users orient between multiple open panels.
- `onClose`: optional callback for host UIs to remove the panel from layout state.
- `onMinimize`: optional callback to collapse panels into a tray or dock.

`OrderFormDrawer` also supports:

- `open`: controls visibility of the drawer.
- `seedRows`: optional `OrderGuideRow[]` to hydrate draft POs generated by the order guide.

## Quick Start Flow

1. Render `OrderGuidePanel` inside your workspace. When users select rows and click ÔÇťSend to Order FormÔÇŁ call your handler with the emitted rows.
2. Show `OrderFormDrawer` (`open={true}`) and forward the rows to its `seedRows` prop. The drawer will generate draft POs and let users edit lines, notes, and status.
3. Once a PO is submitted, display `ReceivingPanel` so receivers can capture delivered quantities. It records lots and ledger entries automatically.
4. Expose `InventoryLotsPanel` for visibility into on-hand inventory and expirations.
5. Expose `StockLedgerPanel` for finance and culinary teams to audit all stock movements.

## Development Notes

- All external integrations are isolated behind adapter functions in `services/adapters.ts`.
- Fixtures are used for development; replace with real data sources and remove `fixtures.ts` once integrated.
- The module exposes the following public surface via `api/purchrec.client.ts`:
  - `fetchOrderGuide()`
  - `savePurchaseOrder()`
  - `submitPurchaseOrder()`
  - `receivePurchaseOrderApi()`
  - `fetchStockLedger()`

Update these adapters as upstream services become available.

## Role Separation & Next Steps

- Purchasing responsibilities (demand modeling, PO drafting/submission) now live exclusively in the Purchasing workspace via `OrderGuidePanel` and `OrderFormDrawer`.
- Receiving responsibilities (dock schedule, lot capture, invoice intake) are isolated to the Receiving workspace; the app header no longer links to ordering from the Receiving menu.
- Inventory visibility (lots + ledger) is available from Purchasing for cross-functional collaboration but remains read-only for receiving roles until access control is wired up.
- Remaining integration tasks:
  - Replace fixture-backed services with real adapters (Neon/Supabase) for menu demand, vendor catalog, and PO persistence.
  - Connect role-based access rules from `AuthContext` to disable UI affordances when users lack Purchasing or Receiving scopes.
  - Wire receiving events to downstream finance workflows (invoice match, variance) once available.
