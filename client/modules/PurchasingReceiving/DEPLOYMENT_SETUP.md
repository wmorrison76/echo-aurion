## Environment Setup

### 1. Create `.env.local` from template

```bash
cp .env.example .env.local
```

### 2. Fill in required variables

**Minimum for production:**

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
VITE_API_URL=https://api.echo-ops.com
```

### 3. Test environment configuration

```bash
# Build with environment variables
pnpm build

# Check for build errors
echo $? # Should be 0

# Run tests
pnpm test
```

### 4. Secret management

**DO NOT commit `.env.local` to git**

Add to `.gitignore`:

```
.env.local
.env.*.local
```

For CI/CD, set environment variables in:

- **Netlify**: Site settings → Build & deploy → Environment
- **Vercel**: Settings → Environment Variables

## Deploying to Netlify

### Step 1: Connect Git Repository

1. Go to https://app.netlify.com
2. Click "New site from Git"
3. Choose GitHub/GitLab/Bitbucket
4. Select your repository
5. Configure build settings (should auto-detect):
   - Build command: `pnpm build`
   - Publish directory: `dist/spa`
   - Functions directory: `netlify/functions`

### Step 2: Configure Environment Variables

1. Go to **Site settings → Build & deploy → Environment**
2. Click **Edit variables**
3. Add all production environment variables:
   ```
   VITE_SUPABASE_URL=<value>
   VITE_SUPABASE_ANON_KEY=<value>
   VITE_API_URL=https://your-domain.com
   ... (add all others)
   ```

### Step 3: Connect Custom Domain

1. Go to **Site settings → Domain management**
2. Click **Add domain**
3. Enter your domain
4. Follow DNS configuration instructions
5. Update DNS records:
   ```
   CNAME: your-site.netlify.app
   ```

### Step 4: Enable HTTPS

1. Go to **Deploys → Deploy settings → HTTPS**
2. Verify SSL certificate is provisioned
3. Force HTTPS: Enable in settings

### Step 5: Deploy

1. Push code to main branch
2. Netlify automatically builds and deploys
3. Monitor build logs in **Deploys** section

**First deployment takes 5-10 minutes**

### Verify Deployment

```bash
# Test HTTPS
curl -I https://your-domain.com

# Check response headers
curl -I https://your-domain.com | grep X-Content-Type-Options
```

## Deploying to Vercel

### Step 1: Connect Git Repository

1. Go to https://vercel.com
2. Click "New Project"
3. Select your Git provider
4. Choose your repository
5. Configure project settings

### Step 2: Configure Build Settings

Vercel should auto-detect:

- **Framework**: Vite
- **Build command**: `pnpm build`
- **Output directory**: `dist/spa`

If not, set manually:

```bash
# In Vercel UI: Project Settings → Build & Development
Build Command: pnpm build
Output Directory: dist/spa
Install Command: pnpm install
```

### Step 3: Set Environment Variables

1. Go to **Settings → Environment Variables**
2. Add production variables:
   ```
   VITE_SUPABASE_URL=<value>
   VITE_SUPABASE_ANON_KEY=<value>
   ... (add all others)
   ```
3. Make sure variables are available for Production environment

### Step 4: Configure Custom Domain

1. Go to **Settings → Domains**
2. Click **Add**
3. Enter your domain
4. Choose domain registration method:
   - **Vercel**: Let Vercel manage DNS
   - **Custom**: Update DNS at your provider
5. Follow verification steps

### Step 5: Deploy

1. Push to main branch
2. Vercel automatically builds and deploys
3. Monitor build logs in **Deployments** section

**First deployment takes 3-5 minutes**

## Supabase Configuration

### Step 1: Create Supabase Project

1. Go to https://supabase.com
2. Click **New Project**
3. Select organization
4. Project name: `echo-ops`
5. Database password: (secure, random)
6. Region: (select closest to users)
7. Click **Create new project**

### Step 2: Wait for Provisioning

- Project takes 1-2 minutes to create
- You'll see "API docs" when ready
- Database is automatically initialized

### Step 3: Get Connection Details

1. Go to **Settings → API**
2. Copy:
   - Project URL → `VITE_SUPABASE_URL`
   - Anon Key → `VITE_SUPABASE_ANON_KEY`
3. Store safely (not in git)

### Step 4: Run Database Migrations

```bash
# Using Supabase CLI (optional, for advanced setup)
npm install -g @supabase/cli
supabase link --project-ref your-project-id
supabase db push

# Or manually in Supabase SQL Editor:
# 1. Go to SQL Editor in Supabase dashboard
# 2. Create new query
# 3. Paste contents of migrations/ folder
# 4. Execute
```

### Step 5: Configure Authentication

1. Go to **Authentication → Providers**
2. Enable desired providers:
   - Email (default, always enabled)
   - Google
   - GitHub
   - Others as needed

3. Configure email templates:
   - Go to **Auth → Email Templates**
   - Customize confirmation email
   - Test sending

### Step 6: Test Connection

```bash
# In Node.js console
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  'https://your-project.supabase.co',
  'your-anon-key'
)

// Try creating a test user
await supabase.auth.signUp({
  email: 'test@example.com',
  password: 'test123456'
})
```

## Integration Setup

### QuickBooks Online

#### Prerequisites

- QuickBooks Online account
- Admin access to add integrations

#### Setup Steps

1. **Create OAuth App**
   - Go to https://developer.intuit.com
   - Create app → Web application
   - Set Redirect URL: `https://your-domain.com/oauth/quickbooks`
   - Copy Client ID and Client Secret

2. **Add Integration to Echo Ops**
   - Admin → Integrations → Add Integration
   - Select: QuickBooks Online
   - Paste Client ID
   - Paste Client Secret
   - Click "Test Connection"
   - Authorize when redirected to QuickBooks

3. **Configure Sync**
   - Go to Integrations → QuickBooks
   - Select entities to sync:
     - [ ] Invoices
     - [ ] Expenses
     - [ ] Chart of Accounts
   - Set sync interval: Hourly/Daily
   - Enable auto-sync

#### Verify

```
Integrations → QuickBooks → Test Sync
Should see: "Successfully synced X invoices"
```

### NetSuite

#### Prerequisites

- NetSuite account with API access
- OAuth credentials

#### Setup Steps

1. **Enable Web Services**
   - Setup → Company → Enable Features
   - Check: "Web Services"
   - Save

2. **Create OAuth Token**
   - Setup → Integration → OAuth 2.0 Tokens
   - Create new token
   - Note Token ID and Token Secret

3. **Add to Echo Ops**
   - Admin → Integrations → Add Integration
   - Select: NetSuite
   - Account ID: (from your NetSuite URL)
   - Client ID: (from OAuth token)
   - Client Secret: (from OAuth token)
   - Click "Test Connection"

4. **Configure Mappings**
   - Integrations → NetSuite → Configure
   - Map inventory items
   - Set subsidiary
   - Enable sync

### SendGrid

#### Prerequisites

- SendGrid account
- API key

#### Setup Steps

1. **Get API Key**
   - SendGrid → Settings → API Keys
   - Create new key (Full Access)
   - Copy key value

2. **Add to Echo Ops**
   - Admin → Integrations → Add Integration
   - Select: SendGrid
   - Paste API Key
   - From Email: noreply@your-domain.com
   - From Name: Echo Ops
   - Click "Test Connection"

3. **Configure Notification Types**
   - Integrations → SendGrid → Configure
   - Enable:
     - [ ] Invoice notifications
     - [ ] Low inventory alerts
     - [ ] Daily digest
   - Set recipient emails
   - Customize templates (optional)

#### Send Test Email

```
Integrations → SendGrid → Send Test Email
Should receive in inbox within 1 minute
```

## Monitoring & Alerting

### Option 1: Sentry (Error Tracking)

#### Setup

1. **Create Sentry Account**
   - Go to https://sentry.io
   - Sign up
   - Create organization

2. **Create Project**
   - Create Project → Select "React"
   - Project name: echo-ops
   - Team: (your team)

3. **Get DSN**
   - Go to Settings → Projects → Client Keys (DSN)
   - Copy DSN

4. **Add to Environment**

   ```
   VITE_SENTRY_DSN=https://key@sentry.io/12345
   VITE_SENTRY_ENVIRONMENT=production
   ```

5. **Test**
   - Trigger an error in production
   - Should appear in Sentry dashboard within 1 minute

### Option 2: Built-in Monitoring

Echo Ops includes built-in monitoring dashboard:

1. Go to **Monitoring** in your instance
2. View:
   - Response times
   - CPU/Memory usage
   - Error rates
   - System health
3. Alerts configured for:
   - High memory usage (>80%)
   - High error rate (>1%)
   - Slow responses (>300ms)

## Security Configuration

### SSL/TLS

- [ ] Verify HTTPS certificate is valid
- [ ] Force HTTPS (Netlify/Vercel automatically does this)
- [ ] Test with: https://www.ssllabs.com/ssltest/

### API Security

- [ ] Enable CORS only for your domain
- [ ] Set secure headers (Content-Security-Policy, etc.)
- [ ] Rate limiting enabled

### Database Security

- [ ] Enable Supabase Row Level Security (RLS)
- [ ] Create appropriate RLS policies
- [ ] Test by trying to access data as different users

### User Security

- [ ] Enable 2FA requirement
- [ ] Set session timeout to 1 hour
- [ ] Enable IP whitelisting (optional)
- [ ] Configure password requirements

### Secrets Management

- [ ] Use environment variables (not hardcoded)
- [ ] Rotate API keys every 90 days
- [ ] Don't commit `.env.local`
- [ ] Use secure password manager for team

## Troubleshooting

### Build Failures

**Problem**: Build fails with "out of memory"
**Solution**:

```bash
# Increase Node memory
NODE_OPTIONS="--max-old-space-size=4096" pnpm build
```

**Problem**: TypeScript errors on deploy
**Solution**:

```bash
# Check locally first
pnpm typecheck

# Fix issues
pnpm format.fix
pnpm build
```

### Deployment Issues

**Problem**: "Cannot find netlify.toml"
**Solution**: Ensure netlify.toml is in root directory, committed to git

**Problem**: Deployment succeeds but site is blank
**Solution**:

- Check that `dist/spa` directory exists
- Verify `publish` directory in netlify.toml
- Clear cache: Settings → Danger zone → Clear cache and deploy

### Integration Issues

**Problem**: QuickBooks sync failing
**Solution**:

1. Check API credentials in admin panel
2. Verify OAuth token hasn't expired
3. Check QuickBooks system status
4. Review error logs in Integrations → Logs

**Problem**: Emails not sending via SendGrid
**Solution**:

1. Verify API key is valid
2. Check from email is verified
3. Review SendGrid logs for bounces
4. Check firewall allows SendGrid IPs

### Performance Issues

**Problem**: Slow page loads
**Solution**:

1. Check database query performance
2. Enable caching layer
3. Review Monitoring dashboard
4. Check for memory leaks in browser console

**Problem**: High CPU usage
**Solution**:

1. Review running processes
2. Check for infinite loops in code
3. Optimize database queries
4. Scale resources if needed

## Rollback Procedure

If something goes wrong in production:

### Quick Rollback (< 5 minutes)

**Netlify**:

1. Go to Netloys → Deployments
2. Find previous stable deployment
3. Click **Publish deploy**

**Vercel**:

1. Go to Deployments
2. Find previous stable deployment
3. Click ⋮ menu → Promote to Production

### Manual Rollback (if needed)

```bash
# Revert last commit
git revert HEAD

# Or checkout previous tag
git checkout v1.0.0

# Rebuild and deploy
git push origin main
```
